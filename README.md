# 2026 최신동향

웹캠 프레임 변화량만 로컬에서 계산해, 움직임이 감지되면 업무용 리포트 화면으로 자동 전환하는 장난형 프라이버시 웹앱 MVP.

## 조사한 오픈소스 단서

- `jasonmayes/JS-Motion-Detection`: 브라우저 웹캠 + 프레임 차이 기반 모션 감지. 별도 AI 없이 가볍게 가능.
- `jimCresswell/web-cam-motion-detection`: `getUserMedia` + Canvas 기반 단순 모션 감지 예시.
- `@tensorflow-models/coco-ssd`: 사람 객체 감지가 가능하지만 초판에는 과함. 얼굴/사람 식별 느낌이 강해져 개인정보 리스크도 커짐.

## 현재 MVP 범위

- 브라우저 `getUserMedia()`로 웹캠 접근
- Canvas에 저해상도 프레임을 그려 픽셀 차이량 계산
- 감도 슬라이더 + 안전/주의/전환 점수 패널
- 감지 영역 선택: 전체 화면, 왼쪽 복도, 오른쪽 복도, 상단/뒤쪽 배경
- 위장 페이지 12종
- 비상 전환 테스트 버튼
- 데모 시나리오 시작: 3초 카운트다운 후 위장 화면 전환
- 지정 HTTPS URL 열기 모드: GitHub/arXiv/Scholar/KOSIS 등 사용자가 지정한 실제 자료 사이트로 현재 탭 이동
- `Ctrl + Shift + L` 단축키 전환/복귀
- 서버 업로드/저장 없음
- 외부 스크립트/CDN 없음
- 외부/GitHub 코드 반입 전 `docs/external-code-intake.md` 체크리스트로 보안 검수

## 실행

```bash
cd /Users/eung/Desktop/koodasai/latest-trends-2026
python3 -m http.server 4180 --bind 127.0.0.1
```

브라우저에서 `http://127.0.0.1:4180` 접속.

## 실사용 튜닝

- 먼저 `비상 전환 테스트`로 원하는 위장 페이지가 자연스러운지 확인.
- 친구/영상 데모는 `데모 시나리오 시작`을 눌러 3초 카운트다운 → 자동 전환 → `생존 성공` 문구까지 한 번에 확인.
- 카메라 권한이 막히면 주소창 카메라 아이콘에서 허용으로 바꾼 뒤 `보호 시작` 재시도.
- 뒤쪽 사람만 감지하고 싶으면 `상단/뒤쪽 배경`을 선택하고 기준 16~24%부터 튜닝.
- 오작동이 많으면 팀장 감도를 둔감한 쪽으로 올리고, 반응이 늦으면 예민한 쪽으로 낮추기.

## 실제 자료 URL 모드

- 전환 방식을 `지정 URL 열기`로 바꾸면 감지/데모 시 현재 탭이 사용자가 지정한 실제 사이트로 이동.
- `https://` 주소만 허용. `http:`, `javascript:`, `data:`, `file:` 같은 주소는 차단.
- 기본 추천 URL: GitHub Trending, arXiv, Google Scholar, KOSIS.
- 실제 사이트 내용은 긁거나 저장하지 않고 브라우저가 직접 이동함.

## 친구에게 보내는 사용법

로컬 서버만 켜서 보여주기:

1. 터미널에서 `python3 -m http.server 4180 --bind 127.0.0.1` 실행.
2. 브라우저로 `http://127.0.0.1:4180` 접속.
3. 위장 페이지를 `GitHub 저장소 화면`, `통계자료 대시보드`, `Excel 통계 시트`, `영어 논문 PDF` 등으로 고르기.
4. `데모 시나리오 시작`을 누르고 3초 카운트다운 뒤 자동 전환되는지 보여주기.
5. 실제 카메라 테스트는 친구 기기에서 직접 권한 허용 후 진행. 서버 업로드/저장 없음.
